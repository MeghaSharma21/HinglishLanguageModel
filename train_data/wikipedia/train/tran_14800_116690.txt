 software pareekshan ek anubhavjanya khoj he, jiske tahat hitdharakon ko pareekshanadhin utpaada yaa seva kii gunvattta ke bare main, us sandarbh main jaankaree uplabdh karai jaati he, jahaan ise prayog ke liye niyat kiya gaya he. software pareekshan, udyog ko software ke karyanvayan main jokhim ko samajhane or saraahna karne kii anumati dene ke liye, software kaa uddeshy or swatantra avlokan bhi pradaan karata he. test takneekon main shaamil he, lekin itane tak hi simit nahin, software bag khojane ke iraade se ek program yaa anuprayog ke nishpaadan kii prakriya.
 yah bhi kaha ja saktaa he ki software pareekshan vah prakriya he, jo yah vidhimaanya or satyapit karti he ki ek software program/anuprayog/utpaada:
 karyrat pareekshan paddati ke aadhaar par, software pareekshan ko vikaas kii prakriya main kisi bhi samay lagu kiya ja saktaa he. baharhaal, test ke adhikansh prayaas tab shuru hote hai, jab aavashyaktaaon ko paribhaashit kar diya gaya ho or koding prakriya purna ho gai ho. vibhinn software vikaas model, vikaas kii prakriya main pareekshan prayaas ko vibhinn charnon par kendrit karte hai. ek adhik paaramparik model main, pareekshan ke adhikansh prayaas tab shuru hote hai, jab aavashyaktaaon ko paribhaashit kar diya gaya ho or koding prakriya purna ho gai ho. apekshaakrit nae vikaas model, jaise Agile yaa XP, aksar vikaas sanchalit pareekshan kaa prayog karte hai or vikaas kii prakriya main pareekshan kaa zyada hissaa, develper ko saunpte hai.
 pareekshan, puri tarah se software ke bheetar sabhi dosha kii pahchaan nahin kar saktaa he. iske bajaay, yah ek aalochana yaa tulana prastut karata he, jo pramanik- siddhant yaa vyavastha ke prati, jiske dwaara ek vyakti kisi samasya ko pahchaan saktaa he - utpaada kii sthiti or vyavhaar kii tulana karata he. in pramaniktaaon main shaamil ho sakate hai vinirdesan, anubandh, tulaneey utpaada, usee utpaada kaa pichhlaa sanskaran, niyat yaa apekshit uddeshyon kaa anumaan, upyogkarta yaa graahak kii apekshaayen, upyukt maanak, prayojya niyam, yaa anya maandand.
 har software utpaada ke lakshith darshak hote hai. udhaaharan ke liye, video game software ke darshak, banking software se puri tarah se alag he. isliye, jab ek sanghathan kisi software utpaada ko viksit athwa usamen nivesh karata he, to vah yah akalan kar saktaa he ki software utpaada apane antim upyogkartaaon, apane lakshith darshakon, apane kharidaaron or anya hitdharakon ko sweekaarya hoga yaa nahin. software pareekshan is moolyankan ke prayaas kii prakriya he.
 2002 main NIST dwaara kiye gaye ek adhyayan se yah pataa chalata he ki software bag se america arthvyavastha ko salanaa $59.5 bilian kii chapat lagati he. yadi behatar software pareekshan kii jaae, to is laagat kaa ek tihaai hissaa bachaaya ja saktaa he.
 prarambh main pareekshan se dibging ke pruthakkaran ko glenford je. myrs dwaara 1979 main pravartit kiya gaya. haalanki unka dhyaan bracoge pareekshan par tha isane software ejineering samudaay kii buniyadi vikaas gatividhiyon, jaise dibging ko satyaapan se alag karne kii ichha kii pushti kii. dev gelprin or wiliam si. hetzel ne 1988 main software pareekshan main pravasthaon or lakshyon ko nimnlikhit charnon main vargeekrut kiya he:
 pareekshan kaa ek praathmik uddeshy, software vifaltaaon kaa pataa lagaana he, taaki dosha ko khoja or sudhara ja sake. yah ek gaiyr-naganya khoj he. pareekshan yah sthapit nahin kar saktaa ki ek utpaada sabhi paristhitiyon main thik-thik kaary kar rahaa he, par sirf itnaa sthapit kar saktaa he ki yah kin vishisht paristhitiyon main thik-thik kaam nahin karata he. software pareekshan ke kaaryakshetra main aksar code kii pariksha ke alaava vibhinn vaataavaran or paristhitiyon main us code kaa nishpaadan, saath hi, us code ke pahaluon kii jaanch shaamil he: kya yah vah kaary karata he jo ise karnaa chahie or kya yah, vah karata jo ise karne kii zarurat he. software vikaas kii vartmaan samskruti main ek pareekshan sanghathan, vikaas dal se alag ho saktaa he. pareekshan dal ke sadasyon ke liye vibhinn bhoomikaayen hotin hai. software pareekshan se praapt jaankaree kaa prayog, us prakriya ko sahi karne ke liye kiya ja saktaa he, jiske dwaara software kaa vikaas kiya gaya he.
 functional pareekshan us pareekshan se sambandh rakhata he, jo code kii ek vishisht karrwai yaa prakaarya ko satyapit karata he. ye aam taur par code kii aavashyaktaaon ke pralekhan main paaya jaataa he, haalanki kuch vikaas prakriya, prayukt maamle yaa upyogkarta khabaron se kaam karte hai. functional test is sawaal kaa jawaab dete hai ki "kya upyogkarta ise kar saktaa he" yaa "kya yah vishesh suvidha kaam karti he".
 non-functional pareekshan, software ke un pahaluon ko darshata he, jo sambhav he ki kisi vishesh prakaarya yaa upyogkarta kii kriya, jaise arohan yaa suraksha se sambandhit naa hon. non-functional pareekshan is tarah ke sawaalon kaa jawaab detaa he ki "ek baar main kitne log log in kar sakate hai", yaa "is software ko hack karnaa kitna aasaan he".
 software ke sabhi dosha, code kii trutiyon ke kaaran nahin hote hai. mahange dosha kaa ek aam strot, apekshaaon ke interval ke kaaran panapata he, udhaaharan he, aparichit apekshaayen, jo program diziner dwaara vilopan kii trutiyon main falit hote hai. apeksha interval kaa ek aam strot, gair-kaaryaatmak apeksha he, jaise pareekshan-kshamta, arohan-kshamta, anurakshan-kshamta, upyog-kshamta, nishpaadan or suraksha.
 software dosha nimnlikhit prakriyaaon ke madhyam se hote hai. programmer ek truti karata he, jo software ke sorce code main ek karaabi main falit hota he. yadi is karaabi ko nishpadit kiya jaataa he, to kuch khaas sthitiyon main system trutipurn parinaam degaa, jo ek vifaltaa kaa kaaran banegaa. sabhi dosha aavashyak rup se vifaltaa main parinat nahin honge. udhaaharan ke liye, ded code main dosha, kabhi vifaltaa main parinat nahin honge. jab parivesh badla jaataa he, tab ek karaabi vifaltaa main badal sakati he. pariveshgat in parivartanon ke udaahranon main shaamil hai, software kaa ek nae hardware platform par chalaaya janaa, sorce data main parivartan yaa bhinn software ke saath parsparik kriya. ek ekal truti, vifaltaa ke vistrut lakshan main parinat ho sakati he.
 aam taur par yah mana jaataa he ki ek karaabi ko jitna pehle khoja jaae, use thik karnaa utana hi sasta hota he. nimnlikhit talika, karaabi ke pataa lagaae jane vaale charan ke aadhaar par, use thik karne kii laagat ko darshaati he. udhaaharan ke liye, yadi apekshaaon main koi samasya sirf release ke baad saamne aati he, to iskaa kharchaa apekshaaon kii samiksha dwaara hi pataa laga liye jane ke kharche se 10-100 guna adhik hoga.
 software vifaltaa kaa ek aam kaaran, anya anuprayog, ek nae oparating system, yaa, badhate hue web brouzar sanskaran ke saath sangatta he. pasch sangatta kii kami ke maamle main, yah isliye ho saktaa he, kyonki programmer ne apane program kii koding yaa software kaa pareekshan sirf, is-or-us oparating system "ke naveenatam sanskaran" par karne kaa vichaar kiya. is tathy kaa anayas nateeja yah he ki: unka naveenatam kaary, software/hardware ke purv mishran ke saath puri tarah se sangat nahin bhi ho saktaa he, yaa yah ek anya mahatvapurn oparating system ke saath puri tarah se sangat nahin ho saktaa he. kisi bhi sthiti main, ye matbhed, ve jo bhi hon, software vifaltaa main parinat hue honge, jaisaa ki computer prayoktaaon ke kuch mahatvapurn tabake ne mahasoos kiya.
 ise ek "suraksha unmukh ranneeti" maan sakate hai, jo ki dev gelprin or wiliam si. hetzel dwaara sujhaaye naveenatam parikshan charan ke saath sateek baithta he, jaisaa ki neeche uddhrut he..
 software pareekshan ke saath ek mukhya samasya he ki input or pre-condishan ke saare combination ke antargat pareekshan sambhav nahin he, yahan tak ki ek saamaanya utpaada ke saath bhi nahin. iskaa matlab he ki software utpaada main dosha kii sankhya kaaphi adhik ho sakati hai or jo dosha kabhi-kabhi hote hai unhen pareekshan main khojna mushkil he. adhik mahatvapurn rup se, gunvattta ke gair-kaaryaatmak aayam - upyog-kshamta, arohan-kshamta, nishpaadan, sangatta, vishwasaneeyata-atyant vyaktiparak ho sakate hai; kuch aisa, jo ek vyakti ke liye paryaapt mulya kaa gathan karata he, jo duusare ke liye asahneeya he.
 software pareekshan ke liye kai drushtikon hai. samiksha, aar-paar guzaarana, yaa nirikshan ko static pareekshan mana jaataa he, jabaki die gaye ek pareekshan maamle ke set ke saath vaastavik nishpadit program code dinemic pareekshan mana jaataa he. static pareekshan ko choda ja saktaa he. dinemic pareekshan tab prayukt hota he, jab khud program ko hi pehali baar prayog kiya ja rahaa ho . dinemic pareekshan, program ke 100% purna hone se pehle code ke vishesh hisson ke pareekshan ke liye shuru ho sakati he . iske liye vishisht takanic hai stabs/drivers kaa prayog karnaa yaa phir ek dibgar parivesh se nishpaadan. udhaaharan ke liye, spredshit program, khud apni prakruti dwaara, kaaphi had tak paraspar rup se jaanche jate hai, jiske tahat parinaam, prayek gananaa yaa text parichaalan ke turant baad pradarshit hote hai.

 haalanki vivaadaaspad, software pareekshan ko software quality ashurence prakriya ke ek mahatvapurn hisse ke rup main dekha ja saktaa he. SQA main, software prakriya visheshgy or lekha pareekshak, software or uske vikaas par ek vyapak drushtikon rakhte hai. ve software ejineering prakriya kii jaanch karte hai or pradutt software main maujuud dosha: tathakthith dosha dar kii maatra ko kam karne ke liye use hi badal dete hai.
 ek "sweekaarya dosha dar" kaa gathan karne vaali chizein software kii prakruti par nirbhar karti hai. udhaaharan ke liye, ek arched video game main, jise ek hawaii jahaaj udaane kaa abhas dene ke liye dizine kiya gaya he, sambhawatah dosha ke prati apekshaakrit adhik sahansheeltaa hogi, us mishan critical software kii tulana main, jiskaa upyog ek airliner ko niyantrit karne main hota he, jo vastav main ud rahaa he.

 software pareekshan, ek computer program ke apekshit parinaamon ko, die gaye ek input set ke liye, iske vaastavik parinaamon se tulana karte hue, software main dosha kaa pataa lagane ke uddeshy se kiya gaya kaary he. virodhabhas swaroop, QA, darasal hone vaali trutiyon ko rokane ke uddeshy se nitiyon or prakriyaaon kaa karyanvayan he.
 software pareekshan tareeqon ko paramparaagat rup se blike box pareekshan or white box pareekshan main vibhajit kiya gaya hai. in donon abhigmon kaa prayog us drushtikon ke varnan ke liye kiya jaataa he, jo ek test ejeeniyar test maamalon kii digining ke liye apanaataa he.
 blike box pareekshan, software se ek "blike box" ke rup main vyavhaar karata he - bina kisi aantarik karyanvayan kii jaankaree ke blike box pareekshan tareekon main shaamil hai: equivelens partishning, boundry vailu enelysis, ol peyars pareekshan, faj pareekshan, model-besd pareekshan, tresebility metrics, exploretary pareekshan or spacification-aadhaarit pareekshan.
 isliye, blike box pareekshan main ek ore "ek asambaddh raay" kaa phayda he or duusari ore "andhi talash" kaa nuksaan.
 white box pareekshan tab hoti he jab pareekshak ke paas, aantarik data sanrachanaon or ise lagu karne vaale code sahit elgoridam tak pahunch sulabh hoti he.
 ve donon ek code kawarej metric ko lautaate hai, jise pratishat ke rup main mapa jaataa he.
 grey box pareekshan main shaamil he aantarik data sanrachanaon or elgoridam tak, pareekshan maamalon ko dizin karne ke liye pahunch, lekin upyogkarta, yaa blike box str par pareekshan. input data kaa parivartan or output ko formet karnaa, grey box ke tahat nahin aata, kyonki input or output us "blike-box" se spasht rup se baahar he, jise ham pareekshan ke tahat system kah rahe hai. yah antar vishesh rup se tab mahatvapurn he, jab do alag developers dwaara likhe code ke do modulon ke bich integration pareekshan sampadit kii jaati he, jahaan pareekshan ke liye sirf interphes ko prastut kiya jaataa he. haalanki, ek data bhandar ko sanshodhit karnaa zarur grey box ke antargat aata he, chunki aam taur par upyogkarta pareekshan ke tahat system ke baahar data badalane main sakasham nahin hoga. grey box pareekshan main reverse ejineering bhi shaamil ho sakati he, udhaaharan ke liye, boundari vailu yaa truti sandesh nirdhaarit karne ke liye.
 pareekshanon ko aksar, software vikaas prakriya main unke shaamil hone kii jagah ke aadhaar par vargeekrut kiya jaataa he, yaa pareekshan kii vishistataa ke str dwaara.
 unit pareekshan un pareekshanon ko sanderbhit karata he, jo code ke ek vishesh khand kii kaaryashiltaa, aam taur par prakriya str par, satyapit karte hai. ek object-unmukh parivesh main, yah aksar shrenee str par hota he or nunatam ikai pareekshan main shaamil hota he constructor or distractor.
 is prakaar ke pareekshan aam taur par developers dwaara likhe jate hai, jab ve code par kaam kar rahe hote hai, yah sunishchit karne ke liye ki ek vishesh prakriya apekshanuroop thik dhang se kaam kar rahi he. code main corner caces yaa anya shakhaon ko pakadaney ke liye, ek prakriya main kai pareekshan ho sakate hai. unit pareekshan akele, software ke ek ansh kii kaaryakshmata ko satyapit nahin kar sakati, balki iskaa prayog yah sunishchit karne ke liye hota he ki software dwaara prayukt building block, ek duusare se swatantra rup se kaary karte hai.
 integration pareekshan kisi bhi prakaar kaa software pareekshan he, jo ek software dizin ke prati ghatakon ke bich interphes ko satyapit karne kaa prayaas karata he. software ghatak ko, punraritiya tareeqe se yaa ek saath ekikrut kiya ja saktaa he. aam taur par pehle vaale tareeqe ko ek behatar abhyaas mana jaataa he, chunki yah interfes muddon ko twarit rup se sthaneey hone or thik hone kii anumati detaa he.
 integration pareekshan, interphes main trutiyaan ujagar karne or ekikrut ghatak ke bich parsparik kriya ko darshane ke liye kaam karata he. uttarottar, architektural dizin ke tatwon ke anuroop jaanche gaye software ghatakon ke bade samoohon ko ekikrut or tab tak jaancha jaataa he, jab tak software ek system ke rup main kaam naa karne lage.
 system pareekshan yah satyapit karne ke liye ek puri tarah se ekikrut system kaa pareekshan karata he ki vah apni aavashyaktaaon ko puraa karata he.
 system integration pareekshan yah pushti karata he ki ek system, system aavashyaktaaon main paribhaashit kisi bhi bahari yaa anya paksh ke system se ekikrut he.
 regration pareekshan ek pramukh code parivartan ke baad dosha ko khojane par dhyaan kendrit karata he. vishesh rup se, yah software pratigaman ko yaa puraane bag jo waapas aa gaye hai, unhen ujagar karne kaa prayaas karata he. jab bhi software prakriya, jo pehle sahi dhang se kaam kar rahi thi or ab apekshanusaar kaary karnaa band kar de, tab aise pratigaman ghatit hote hai. vishisht rup se, pratigaman, program parivartan ke ek anpekshit parinaam ke rup main ghatit hote hai, jab software kaa nav viksit hissaa, pehle se maujuud code ke saath takaraata he. regration pareekshan ke aam tareekon main shaamil he, pehle chalaae gaye pareekshan ko phir se chalaanaa or jaanch karnaa ki pehle thik kii gai kharabiyaan kahin phir se ubhar to nahin ai hai. pareekshan kii gaharaai, jari prakriya main charan par nirbhar karti he or atirikt visheshtaon ke jokhim par. ve yaa to purna ho sakate hai, release main kaaphi der se jode gaye parivartanon ke liye yaa phir jokhim bhare, bahut uthle tak, prayek suvidha par sakaaratmak pareekshan se bane hue, agar badlaav, relies main jaldi ho rahe hai yaa kam jokhim vaale samajhe jate hai.

 alpha pareekshan, sambhavit upyogkartaaon/graahakon yaa develper kii site par ek swatantra test team dwaara nakli yaa vaastavik sanchaalan pareekshan he. software ke beta pareekshan main jane se pehle, alpha pareekshan ko aksar of-da-shelf software ke liye aantarik sweekruti pareekshan ke ek rup main prayog kiya jaataa he.
 beta pareekshan alpha pareekshan ke baad aata he. beta sanskaran ke naam se vikhyath software kaa sanskaran, programing team se baahar ek simit graahakon ke liye jari kiya gaya. software ko logon ke samoohon ke liye jari kiya jaataa he, taaki aage ke pareekshan yah sunishchit kar saken ki utpaada main nunatam kharabiyaan yaa bag hai. kabhi-kabhi, beta sanskaran ko bhavi upyogkartaaon kii adhiktam sankhya ko pratikriya ke kshetra ko badhaane ke liye, khuli janata ke liye uplabdh karaayaa jaataa he.
 software ke gair-kaaryaatmak pahaluon ke pareekshan ke liye vishesh tareeqe maujuud hai. kaaryaatmak pareekshan ke vipareet, jo software ke sahi sanchaalan ko sthapit karata he, gair-kaaryaatmak pareekshan is baat kii pushti karata he ki software tab bhi thik dhang se kaary karata he, jab ise avaidh yaa apratyaashit input praapt hoti he. software folt injection, fajing ke rup main, ek gair-kaaryaatmak pareekshan kaa udhaaharan he. gair-kaaryaatmak pareekshan, vishesh rup se software kii khatir, yah sthapit karne ke liye dizine kiya gaya he ki pareekshan ke antargat rachana, avaidh yaa anpekshit input ko sahan kar sakati he yaa nahin or is prakaar ve input vailidation routine or saath hi saath erar-handlling routine kii majbootee ko sthapit kar sakenge. vibhinn vanijyik gair-kaaryaatmak pareekshan upkaran, software folt injection prishth se jude hote hai; kai khule strot ke or muft software upkaran uplabdh hai, jo gair-kaaryaatmak pareekshan karte hai.
 parformens pareekshan, yaa lod pareekshan yah dekhne ke liye jaanch karata he ki software, badi maatra main data yaa upyogkartaaon ko sambhaal saktaa he yaa nahin. aam taur par ise software scalebility ke rup main janaa jaataa he. gair-kaaryaatmak software pareekshan kii yah gatividhi aksar kshamta pariksha ke rup main jaani jaati he.

 anterrashtriyakaran or sthaniyakaran kii jarurat software ke in pahaluon kii jaanch ke liye hoti he, jiske liye ek chhadmasthaniyakaran vidhi kaa istemaal kiya ja saktaa he. yah is baat kii pushti karegaa ki anuprayog, ek nai bhashaa main anuwaad kiye jane yaa ek nai samskruti ke liye anukulit kiye jane ke baad bhi kaam karata he.

 software pareekshan ke ek aam tareeke ko, ise graahak ko bheje jane se pehle or isaki kaaryakshmata viksit kiye jane ke baad, pareekshakon ke ek swatantra samooh dwaara sampadit kiya jaataa he. is abhyaas ke parinamswaroop aksar pareekshan charan kaa, pariyojna main hui deree kii kshatipurti ke liye pariyojna bafer ke rup main upyog kiya jaataa he or is prakaar se pareekshan ke liye samarpit samay ke saath samjhauta hota he. ek or paripati he pariyojna ke shuru hone ke kshan hi software pareekshan ko shuru kar dena or pariyojna ke samaapan tak yah ek satat prakriya he.

 aam taur par, jin vishyon tak gunvattta simit hoti he, ve hai shuddhta, sampoornataa, suraksha, lekin ISO maanak ISO 9126 ke tahat varnit adhik takneekee aavashyaktaaen bhi shaamil ho sakati hai, jaise kshamta, vishwasaneeyata, dakshata, suvahyata, rakh-rakhaav, sangatta or 0}prayojyata.
 kai aam software upaay maujuud hai, jinhen aksar 'matrix' kaha jaataa he, jinka prayog software kii haalat yaa pareekshan kii paryaaptta ke maapan ke liye kiya jaataa he.
 software pareekshan prakriya kai artiffact ko utpann kar sakati he.
 software pareekshakon or gunvattta aashwaasan visheshgyon kii peshewar aakankshaon ko bal dene ke liye kai pramann kaarykram maujuud hai. vartmaan main pradaan kiye jane vaale pramann main aisa koi nahin he, jisamen vastav main aavedak ke liye software pareekshan kii kshamta ko pradarshit karne kii aavashyaktaa hoti he. koi bhi pramanikaran, vyapak rup se sweekrut gyaan ke dhaanche par aadhaarit nahin he. is tathy ne kuch logon ko yah kehane ke liye prerit kiya ki pareekshan kshetra abhi pramann ke liye taiyaar nahin he. akele pramann ek vyakti kii utpaadakta, unke kaushal, yaa vyaavhaarik gyaan ko maap nahin saktaa or ek pareekshak ke rup main unki kshamta, yaa peshevraana style kii guarantee nahin de saktaa.
