 IPv4 adress aamtaur par dot-dashamalav sanketan main darshaae jate hai . prayek bhaag adress ke 8 bit kaa pratinidhitiv karata he or isliye ocotate kahalaaya jaataa he. takneekee lekhan ke kam aam maamalon main, IPv4 adress, shodash aadhari, ashtadhari, yaa binary abhivedan main prastut kiya ja saktaa he. sarvaadhik pratinidhitiv main prayek ocotate vyaktigat rup se badla jaataa he.
 internet protocol ke vikaas ke praarambhik charan main, network ke prashaskon ne IP adress kaa do bhaagon main varnan kiya, network sankhya bhaag or mejabaan sankhya main bhaag. ek adress main sarvocch kram ocotate ko network number abhinihit kiya jaataa tha or baaki ke bits rest field yaa host identiphire kehalaae jate the or network ke abhyantar mezbaan sankhyankan ke liye istemaal kiye jate the. yah paddati jald hi annuroop sabit hui jaise hi atirikt network viksit hue jo maujuuda network se svachhand the, jinhen pehle se hi ek network sankhya dwaara abhinihit kiya hota tha. 1981 main, classphul network sanrachana ke samaveshan ke saath internet parichayan vinirdesh main sanshodhan kiya gaya.
 classphul network dizine ek badi sankhya ke liye vyaktigat network kaary kii anumati detaa tha. ek IP adress ke sabase mahatvapurn ocotate ke pehle teen bit, adress kii shrenee ke rup main paribhaashit kiye gaye. saarvbhaumik unicast parichayan ke liye teen shreneeyaan paribhaashit kii gai. vyutpann varg ke aadhaar par network pahchaan puure pate ke octate seemaa kshetron par aadhaarit tha. prayek shrenee network pahachanakarta main kramshah atirikt octate kaa upyog karti thi, is prakaar se ucch varg shrenee main sambhav mejabaanon kii sankhya kam ho jaati he.nimna talika is faltah aprachlit pranali kaa ek simhavlokan detaa he.
 'sabanetwork' or 'classphul network' lekh main is abhikalp kaa vivaran samajhaayaa gaya he.
 haalanki classphul network abhikalp ek safal vikasaatmak avasthaa thi lekin internet ke betahasha vistaar ke kaaran ise aage badhanaa mushkil ho gaya or jab nae ypi adress khand dene ke liye classles inter-domen rooting kaa srijan kiya gaya or rooting protocol ke liye ipivi4 adress kaa prayog karne ke liye nae niyam taiyaar kiye gaye to ise chhod diya gaya. swecchit-lambaai upsargon kaa aabantan evam anumaargan karwaane ke liye siidiar veriable-lenth sabanet masking par aadhaarit he.
 aaj, classphul network sankalpana ke avshisht keval ek simit kshetra main kaam karte hai jo ki kuch network software or hardware ghatakon ke diphalt kanfigration maapdand hai or network prashaskon kii charchaa ke takneekee jorganon main prayukt hote hai.
 aarambhik network abhikalp main jab sanchaar ke liye sabhi internet prayogkartaaon ke bich vaishvik str par ek chaur se duusare chaur tak jodne kii baat sochi gai to yah lakshya tha ki prayek computer yaa upkaran kaa ek vishisht ypi pataa ho. lekin yah paaya gaya ki hameshaa aisa kar paanaa sambhav nahin he kyonki nijee network viksit ho gaye or saarvajanik paton kaa sthaan sangrakshit karne kii jarurat padi .
 jo computer internet se nahin jude hai, jaise ki kaarkaane kii mashine jo tisipi/ypi ke madhyam se keval aapas main sanchaar karti hai, unke liye vaishvik taur par vishisht ypi pataa hona jaruri nahin he. nijee network ke liye IPv4 paton main se teen simaaen, har varg ke liye ek seemaa, 1918 RFC main arakshit thi. in paton kaa margan internet par nahin kiya jaataa or isliye inkaa prayog ypi paton kii panji ke saath samanvit karne kii aavashyaktaa nahin he.
 aaj, jab jarurat padati he, aise nijee network aamtaur par network pataa anuwaad ke madhyam se internet se conect ho jate hai.
 koi bhi upyogkarta kisi bhi arakshit blaak kaa upyog kar sakate hai. aamtaur par, ek network vyavasthaapak ek khand kaa sabnats main vibhaajan kar detaa he, udhaaharan ke liye, kai gharelu anurmagak swatah 192.168.0.0 - 192.168.0.255 ke ek vitath adress shrenee kaa upyog karte hai.
 ypi sanskaran 4 pate kaa sthaan teji se apane aadhikarik taur par uplabdh abantit pate ke khandon ke samaapan par pahunch rahaa he.
 sangrakshan takneekon ke baavjood, IPv4 adress sthaan ke teji se samaapan ne internet ejineering task force ko internet kii parichayan kshamta kaa vistaar karne ke liye nai praudyogikiyon kaa pataa lagane ke liye prerit kiya. iskaa sthaayee samaadhaan, internet protocol kaa hi ek nayaa swaroop, samajha tha. internet protocol kii yah agali peedi, jiskaa uddeshy internet par IPv4 kii jagah lenaa tha, antat 1995 main ise internet protocol sanskaran 6 kaa naam diya gaya. pataa aakaar 32 se badha kar 128 bit yaa 16 ocotate kar diya gaya, jo, network block ke ek udaar kaam ke saath bhi, nikat bhavishya ke liye paryaapt mana jaataa he. ganitiya, nayaa adress sthaan, adhiktam 2, yaa lagbhag 3.403Ã—10 adwitiya pate kii kshamta pradaan karata he.
 nayaa dizin keval adress kii paryaapt maatra pradaan karne ke lakshya par aadhaar nahin he, balki sabanet anumaargan upsargon ke kushal ekatrikaran ko anumaargan nodes par hone kii anumati dene ke liye. parinamswaroop, anumaargan talika aakaar chota hota hai or sabase chota sambhav vyaktigat aavanton 264 host ke liye ek sabanet he, jo puure IPv4 internet ke aakaar kaa vargakar he. in staron par, vaastavik adress upyog dar kisi bhi IPv6 network khand par chota hoga. nayaa dizin ek network khand ke parichayan buniyadi dhaanche ko alag karne kaa mauka bhi pradaan karata he-jo khand ke uplabdh sthaan kaa sthaneey prashasan he-ek parichayan upsarg se jo ek network ke liye bahari yatayaat ko maarg dene ke liye upyog kiya jaataa he. IPv6 main aisi suvidhaayen he jo, yadi vaishvik sampark yaa anumaargan niti main parivartan ho, to bina aantarik nae swaroop yaa nae sankhyankan ke swatah hi puure network kaa anumaargan upsarg badal detii hai.
 IPv6 paton kii badi sankhya vishisht prayojanon ke liye bade block nirdisht karne kii anumati dete he or jahaan uchit he, kushal maarg ke liye ekatrit hone kii. ek bade adress sthaan ke saath ek jatil pataa sangrakshan takanic kii jarurat nahin hoti, jaise clasles inter-domen rooting main upyog kii jaati he.

 jab ek computer har baar ek hi ai pi adress kaa prayog karne ke liye samanurup kiya jaataa he to vah tej ho jaataa he, ise static IP adress kehete hai. iske vipareet, un sthitiyon main jab computer ke ypi pate swatah hi niyukt kiye jate hai, use dinemic ypi adress ke rup main janaa jaataa he.
 static IP adress ek vyavasthaapak dwaara haath se computer ko saunpa he. isaki sateek prakriya platform ke anusaar pruthak hoti he. yah dinemic ai pi adress se asaadrishy hai, jo yaa to computer interphes yaa swayam host software, jaise xeronkonf main, dwaara nirdisht hote he yaa dinemic host kanfigreshna protocol ke istemaal se ek sarwar dwaara nirdisht kiye hote he. haalanki DHCP ke upyog se nirdisht ypi adress lambey samay ke liye ek jaise rahete hai, vah aam taur par badal sakate hai. kuch maamalon main, ek network vyavasthaapak sakriy rup se nirdisht static IP adress lagu kar saktaa hai. is maamle main, ek DHCP sarwar kaa upyog hota he, lekin ek vishisht computer ko hameshaa ek hi ypi adress nirdisht karne ke liye, iskaa samanurup vishesh rup se kiya jaataa he. yah static IP adress ko mukhya rup se samanurup hone kii anumati detaa he, jisamen network par prayek computer ko hastchalit pranali dwaara vishesh rup se samanurup karne kii aavashyaktaa nahin padati.

 gatisheel ypi pate dinemic host kanfigareshan protocol sarvaron dwaara atyadhik praayah LANs or broadband network par niyukt hote hai. inhen isliye istemaal kiya jaataa he kyunki yah ek network par prayek upkaran ke liye vishisht static adress nirdisht karne ke prashasnic boj se bachaa sakate he. yah kai upkarnon ko ek network par simit adress sthaan main sahabhaagi banane kii anumati bhi detaa he, par keval yadi un main se kuch kisi vishesh samay par online hon. kai vartmaan desktop oparating system main, gatisheel ypi vinyas difolt rup se sakasham kiya hota he taaki ek upyogkarta ko DHCP sarwar ke saath ek network se judane ke liye apane haath se koi settings darj karne kii aavashyaktaa naa pade. gatisheel IP adress nirdisht karne ke liye DHCP hi akeli takanic nahin he. dyaap or kuch broadband network bindu se bindu protocol kii gatisheel adress visheshtaayen upyog karte hai.
 ek stick dinemic IP adress yaa stick ypi cabel or DSL internet adhigam bhugtankarta dwaara ek gatyaatmakataa se niyukt ypi adress jisamen aksar badlaav nahin aata, uskaa varnan karne ke liye istemaal kiya jaataa he. pate aamtaur par DHCP protocol ke saath niyukt kiye jate he. chunki modem aamtaur par vistaarit avadhi ke samay ke liye yantrachaalit hote hai, isliye adress patte aksar lambey samay ke liye niyamit kiye jate hai or samaapti hone par naveekrut kar die jate hai. yadi ek modem band kar diya jaae or adress patte ke agle samaapan se pehle phir se sanchalit kiya jaae, to is baat kii sarvaadhik sambhaavna he ki use vahi IP adress praapt hoga.
 RFC 3330 IPv4 networkon ke liye sthaneey-link adressing main vishesh upyog karne ke liye adress block, 169.254.0.0/16, paribhaashit karata he. IPv6 main, sthitik yaa gatik adress kaaryon kaa upyog karne vaala, prayek interphes fe80::/10 sabanet main swatah ek sthaneey-link adress praapt karata he.

 jab sthaneey-link IPv4 adress block arakshit hua tha, adress swachaalit-samakriti ke tantra ke liye koi bhi maanak astitv main nahin tha. shoonyata bharna, microsoft ne ek karyaanvayan banaaya jise swatah nijee IP adressing kaha gaya. microsoft kii vipanan kshamta ke kaaran APIPA ko lakhon mashiinon par tainaat kiya gaya he or, is prakaar yah udyog main ek vaastavik maanak ban gaya he. kai varshon baad, IETF ne is kaaryashiltaa ke liye ek aupchaarik maanak, RFC 3927, sthaneey-link adress IPv4 ke sakriy vinyas kaa adhikari paribhaashit kiya.
