 software ejineering main, dizine pattern aam taur par software dizine main hone vaali samasya ke liye ek saamaanya pun: prayojya samaadhaan he. ek dizine pattern ek purna dizine nahin he jise sidhe code main badla ja sake. samasya kaa kaise nidaan kiya jaae, iskaa yah ek vivaran yaa khaakaa he jise kai vibhinn sthitiyon main istemaal kiya ja saktaa he. object-unmukh dizine pattern, isamen shaamil antim anuprayog virgon yaa object ko nirdisht kiye bina, aam taur par virgon yaa object ke bich sambandhon or parsparik kriya ko darshaate hai.
 dizine pattern, modul or interconection ke prabhaav kshetra main rahete hai. ucch str par, aise vaastukala pattern maujuud hote hai, jinka vistaar apekshaakrit bada hota hai, jo aam taur par ek puri pranali dwaara anusaran kiye jane vaale ek samgr pattern kaa varnan karte hai.
 sabhi software pattern, dizine pattern nahin hote. udhaaharan ke liye, kalanvidhi, software dizine samasyaon ke bajaay pariklan samasyaon ko sulajhaata he.
 pattern, christopher alexander dwaara ek vaastukala avadharana ke rup main utpann hue. 1987 main, kent back or ward kaningham ne programing par pattern lagu karne ke vichaar ke saath prayog shuru kiya or us varsh OOPSLA sammelan main apane parinaam prastut kiye. baad ke varshon main, bek, kaningham or dusron ne is par kaam karnaa jari rakhaa.
 dizine pattern: eliments of rieushable object-oriented software pustak ke 1994 main prakashit hone ke baad, dizine pattern ne computer vigyaan main lokapriyata hasil kii. usee saal programing kii pattern bhashaaen par pehla sammelan yojit hua or agle varsh portland pattern bhandar ko dizine pattern ke pralekhan ke liye sthapit kiya gaya. is shabdaawalee kaa kaaryakshetra vivaad kaa vishay bana hua he. dizine pattern style kii ullekhneey pustakon main shaamil hai:
 haalanki dizine pattern kaa vyaavhaarik anuprayog ek tathy he, tathaapi dizine pattern kii avadharana ko niyamanishth karne kaa kaary kai varshon tak atka rahaa. 
 dizine pattern, vikaas prakriya ko siddh or parakhe hue vikaas maandand pradaan karte hue tej kar saktaa he. prabhavi software dizine main un muddon par vichaar karne kii aavashyaktaa hoti he, jo karyanvayan main baad tak dikhaai nahin de sakate hai. dizine pattern kaa punarprayog kuch sookshm muddon ko rokane main madad karata he, jo vyapak samasya paidaa kar sakate hai or yah code likhane waalon or vaastukaron ke liye, jo pattern se parichit hai, code pathneeyata main sudhaar karata he.
 lacheelaapan praapt karne ke liye, dizine pattern aam taur par paroksh upaay kaa atirikt str pesh karte hai, jo kuch maamalon main parinami dizine ko jatil or anuprayog pradarshan ko chot pahunchaa sakate hai.
 paribhaasha ke anusaar, ek pattern kaa uskaa upyog karne vaale prayek anuprayog main nae sire se programing kiya janaa chahie. chunki kuch lekhak ise ghatakon dwaara uplabdh karaae gaye anusaar software punahprayog se ek pichde kadam ke rup main dekhte hai, shodhakartaaon ne pattern ko ghatakon main badalane ke liye kaam kiya. meyer or arnot, sarvaadhik gyaat pattern ko ghatak main parivartit karne main do tihaai safalataa dar kaa daawa karte hai.
 aksar log, kuch software dizine takneekon ko kuch samasyaon ke liye kaise lagu karen, bas itnaa hi samajhate hai. in takneekon ko samasyaon kii ek vyapak shrenee ke liye lagu karnaa kathin he. dizine pattern saamaanya samaadhaan pradaan karte hai, jo ek aise praaroop main pralekhith hota he, jise kisi vishisht samasya se bandhe vinirdesan kii aavashyaktaa nahin hoti he.
 dizine pattern kai virgon se rache hote hai . sanrachana, pratibhaagi or sahyog khand vishesh rup se dilachasp hai. ye varg ek dizine akruti kii vyakhya karte hai: ek aadya rup micro-architekchar jise developers copy karte hai or dizine pattern dwaara varnit aavarti samasya ke samaadhaan hetu apane vishisht dizine ke liye anukulit karte hai. micro-architekchar, program ghatakon or unke sambandhon kaa ek set he. developers, is aadya rup micro-architekchar ke apane dizinon main pravartan dwaara dizine pattern kaa upyog karte hai, jiskaa matlab he ki unke dizinon main micro-architekchar kii sanrachana or sanghathan, chaynit dizine akruti ke samaan hoga.
 iske atirikt, pattern, developers ko software kii paraspar kriya ke liye acchhee tarah se gyaat, acchhee tarah se samajhe hue naam ke prayog se samvaad karne kii anumati dete hai. aam dizine pattern ko asthai dizine se adhik majboot banaate hue, samay ke saath sudhara ja saktaa he.
 dizine pattern ko vishesh domen main kodebaddh karne ke prayaas bhi kiye gaye hai, jisamen vartmaan dizine pattern ke upyog sahit domen vishisht dizine pattern shaamil he. udaahranon main shaamil hai prayokta antarfalak dizine pattern, soochna vizualizeshan, "surakshit prayojyata" or web dizine.
 vaarshik pattern langwagees of programing sammelan kii kaarywahi main domen vishesh pattern ke kai udhaaharan shaamil hai.
 dizine pattern ko multah criational pattern, strectural pattern or biheviaral pattern shreniyon main baanta gaya he or unki vyakhya delygation, agrigeshan or kansaltesan kii avadharanaon ke upyog dwaara kii gai he. object-unmukh dizine kii adhik prishthbhumi jaanane ke liye kapaling or kohizan dekhen. object-unmukh programing kii adhik prishthbhumi ke liye, inheritance, interfes or polymorfizam dekhen. ek anya vargeekaran ne architektural dizine pattern kii dharna ko bhi pesh kiya he, jise software ke architekchar str par lagu kiya ja saktaa he jaise ki model-vyuu-controlar pattern.
 dubhashie ke zarie pratinidhitiv ko vyakaran samet paribhaashit karata he.
 vibhinn object main propertiz daalane ke liye milaayaa jaataa he.
 ek dizine pattern ke liye pralekhan, us sandarbh, jisamen is pattern kaa upyog kiya gaya he, sandarbh ke andar ke force jise pattern sulajhane kaa prayaas karata he or prastaavit samaadhaan kii vyakhya karata he. dizine pattern ko pralekhith karne ke liye koi ek maanak praaroop nahin he. balki, vibhinn pattern lekhakon dwaara vibhinn prakaar ke praroopon kaa prayog kiya gaya he. tathaapi, martin foller ke anusaar, kuch nishchit pattern praaroop anya praroopon kii tulana main zyada lokapriy ho gaye hai or falswaroop pattern lekhan ke nae prayaason ke liye ek aam aarambhik charan ban gaye hai. aam taur par istemaal kiye jane vaale pralekhan praaroop kaa ek udhaaharan he, erik gamma, richard hem, ralf johnson or jon licides dwaara unki pustak dizine pattern main prayukt praaroop. isamen nimnlikhit varg hai:
 computer vigyaan ke kshetra main, dizine pattern kii avadharana ke bare main kuch aalochanaayen maujuud hai.
 dizine pattern kaa durupyog, anaavashyak code ko lagu karne main parinat ho saktaa he. yah kushal software vikaas kii saadagi ke pratimaan ke vipareet he.
 
 dinamik programing language ke upyogkartaaon ne kai dizine pattern kii bhashaaon, jaise C++ or Java kii simaaon ke liye varderound ke rup main charchaa kii he. udhaaharan ke liye, vizitor pattern ko us bhashaa main lagu karne kii jarurat nahin he, jo multimethds kaa samarthan karti he. vizitor kaa uddeshy maujuuda virgon main, unhen bina sanshodhit kiye nai sankriyaayein jodna he. C++ main, ek varg ko ek vishisht or band tareekon ke set vaale, sintactic strecchar ke rup main ghoshit kiya gaya he. multimethds vaali ek bhashaa main, jaise kaaman lisp varg ke liye tareeke, us varg sanrachana se baahar hote hai or vyakti unhen bina badale nae tareeke jod saktaa he. isi tarah, decorator pattern, dinemic dailygation ke karyanvayan ke baraabar he, jaisaa ki kaaman lisp, objective C, self or JavaScript main paaya jaataa he.
 peter norwig ne dizine pattern in dinamik programing main dinamik bhashaaon main vibhinn pattern ko karyanvit karne kii ghisi-pity baat kii charchaa karte hai. norwig or anya ne un bhashaa gunon kii vyakhya kii he, jo vibhinn pattern ko aaveshtit yaa sthanapann karte hai, jise C++ ke upyogkarta ko swayan ke liye lagu karnaa jaruri hoga.
 kuch lekhakon kaa aarope he ki dizine pattern, ebstrection ke anya roopon se mahatvapurn rup se bhinn nahin hai or programing ke kshetra main maujuuda tathyavishyak varnan ke liye nai shabdaawalee kaa prayog anaavashyak he. model-vyuu-cantrolar pratimaan ko "pattern" ke ek udhaaharan ke rup main uddhrut kiya gaya he, jo "dizine pattern" kii avadharana se kai varshon pehle se vyaapt he.is par kuch or log tark dete hai ki dizine pattern samudaay kaa praathmik yogadan alexander kii pattern bhashaa kaa istemaal pralekhan ke ek rup main karnaa tha; ek abhyaas jise aksar sahitya main nazarandaaj kiya gaya he.
