 saanchaa:Software development process
 software deveopment prakriya kisi software utpaada ke vikaas main adhiropit ek sanrachana he. iske paryaayavachi shabd software jivan chakra or software prakriya hai. aisi prakriyaaon ke liye kai model hai. inamen se prayek prakriya ke dauraan hone vaale vibhinn prakaar ke kaaryon yaa gatividhiyon tak pahunch kaa varnan karte hai.
 software deveopment sanghathanon kii sabase badi nikaay prakriya sambandhi paddatiyon kaa karyanvayan karti he. unamem se anek raksha udyog main hai, jinke liye sanyukt rajya america main anubandhon ko praapt karne ke liye 'prakriya modelon' ke aadhaar par moolyankan kii aavashyaktaa hoti he.
 chayan kii paddati kaa varnan karne, software ke liye jivan chakra kaa karyanvayan karne or uske nirikshan ke liye antarrashtriya maanak ISO 12207 he.
 utpaadakta or gunvattta main sudhaar karne vaali ek dohraane yogya, purvanumey prakriya kaa pataa lagaana dashakon se lakshya rahaa he. kuch log software lekhan ke prakatat: kathin kaary ko vyavasthit yaa nishchit rup dene kii koshish karte hai. anya log software lekhan ke liye pariyojna prabandhan takanic prayog main laate hai. pariyojna prabandhan ke bina, software pariyojnaon kii supurdagee main sahaj main hi vilamb ho saktaa he yaa isaki laagat bajet se adhik ho sakati he. kaaryaatmakataa, laagat, yaa supurdagee kii samay-saarini ke sambandh main badi sankhya main pariyojnaon dwaara apni apekshaaon ko puraa nahin kar paane ke kaaran, prabhavakaari pariyojna prabandhan main kami rah jaati he.
 sansthaaen ek software ejineering prakriya samooh kii rachana kar sakati hai, jo prakriya main sudhaar kaa centre bindu hota he. vibhinn kaushalon vaale line peshewaron se bana yah samooh software ejineering prakriya main sudhaar laane ke kaary main shaamil prayek vyakti ke sahyogpurn prayaas ke centre main sthit rahata he.
 software utpaada taiyaar karne main mahatvapurn kaary aavashyaktaaon kaa pataa karnaa yaa aavashyaktaaon kaa vishleshan karnaa he. graahak jinhen antim parinaam ke rup main chaahate hai, us sambandh main unke paas vishesh rup se ek amoort vichaar hota he, na ki software ko kya karnaa chahie is sambandh main. is bindu par apurna, aspasht yaa virodhabhasi aavashyaktaaon kii kushal or anbhavi software ejiniyaron ke dwaara pahchaan kii jaati hai. sajiv code kaa pradarshan aksar hi karte rahane se aavashyaktaaon ke galat rahane kaa jokhim kam ho saktaa he.
 ek baar graahakon se saamaanya aavashyaktaaon ko ekatrit kar lene ke baad vikaas ke kaary-kshetra ke vishleshan kaa nirdhaaran kiya janaa chahie or ise spasht rup se vyakt karnaa chahie. ise aksar ek kaary-kshetra sambandhi dastavej kaha jaataa he.
 laagat ke falan ke kaaran yaa vikaas ke aarambh main aspasht aavashyaktaaon ke parinaamon ke kaaran kuch kaaryaatmakataa pariyojna ke kaary-kshetra se baahar ho ja sakati he. yadi vikaas bahya rup se kiya jaataa he, to is dastauez ko ek kanoonee dastavej mana ja saktaa he taaki yadi kabhi koi vivaad ho to graahak ko kiye gaye vaadon ke sambandh main kisi bhi sanshayta ko spasht kiya ja sake.
 ek nae software ko taiyaar karne main aksar domen vishleshan pehla kadam hota he. chaahe yah vartmaan software main koi parivardhan, ek nayaa program, ek nayaa upatantr yaa ek purna nayi vyavastha ho. yah maanakar ki developers naye software ke vishay-kshetra ke kushal jaankaar nahin hai, pehla kaam software ke tathakthith 'domen' kaa pataa lagaana he. domen ke sambandh main ve pehle se hi jitne adhik jaankaar hote hai, utane kam kaam kii aavashyaktaa hoti he. is kaary kaa ek anya uddeshy us kshetra ke visheshgyon ke saath aavashyaktaaon ko praapt karne vaale or use ekatrit karne vaale vishleshkon kaa domen kii apni shabdaawalee main baat karana he jisase in visheshgyon dwaara vyakt kiye gaye kathanon ke sambandh main behatar samaj viksit kiya ja sake. yadi vishleshak uchit shabdaawalee kaa prayog nahin karata he to yah sambhaavna he ki unki baton ko gambhirtha se nahin liya jaayegaa. is prakaar, yah charan aavashyaktaaon kaa pataa karne or use ekatrit karne main ek mahatvapurn prastaavna hogi. yadi kisi vishleshak ne uchit kaam nahin kiya he to bhram utpann ho saktaa he: "main jaanata hum ki aap yah maanate hai ki maine jo kaha uske sambandh main aap jo sochate he use aapane samaj liya, lekin main is sambandh main nishchit nahin hum ki aap yah samajhate hai ki aapane jaisaa suna vah meraa matlab nahin tha ".
 vinirdesh vah kaary he jiske arntagat likhe jane vaale software kaa sambhawatah sahi bhang se varnan kiya jaataa he. vyaavhaarik rup se, sabase safal vinirdeshon ko pehle se hi suviksit prograamon ko samajhane or adhik anukool banane ke liye likhaa jaataa he. yadhyapi aksar program ke vikaas ke purv suraksha kii drushti se atyant mahatvapurn software tantron kaa saavdhaneepurvak vistrut vivaran diya jaataa he. vinirdesh bahari antarfalkon ke liye sabase adhik mahatvapurn hote hai jinhen sthir bana rahanaa chahie. vinirdesh paryaapt rup se sahi hai yaa nahin iskaa nirdhaaran karne kaa ek acha tarika aavashyaktaaon or upyog kii sthitiyon kii shuddh rup se sakshmata sunishchit karne ke liye tisare paksh dwaara dastavejon kii samiksha karana he.
 ek software system yaa software sanrachana kii banaavat us tantra kaa ek amoort nirupan he. sanrachana kaa sambandh yah sunishchit karnaa he ki software system utpaada kii aavashyaktaaon ko puraa karegaa, saath hi saath yah bhi sunishchit karnaa he ki bhavishya main aavashyaktaaon ke sambandh main charchaa kii ja sakati he. sanrachana sambandhi kadam software system or anya software utpaadon ke bich interpheson, saath hi saath antarnihit hardware yaa parichaarak prachaalan system kii charchaa karata he.
 karyanvayan prakriya kaa ek hissaa he jahaan software ejeeniyar pariyojna ke liye code kaa program likhte hai.
 software pareekshan software deveopment prakriya kaa ek abhinn or mahatvapurn hissaa he. prakriya kaa yah hissaa yah sunishchit karata he ki bag kii pahchaan yathashighra kar lee jaye.
 bhavishya main rakrakhaav or software main vruddhi karne ke uddeshy se software ke aantarik dizin kaa dastavejikaran sampoorn vikaas ke dauraan kiya jaataa he. isamen ek API kaa sanlekhan shaamil ho saktaa he, chaahe vah bahari yaa aatanrik ho.
 code kaa uchit rup se pareekshan karne ke baad pariniojan aarambh hota he, ise nirgaman or bikri ke liye sweekrut kiya jaataa he, nahin to kisi utpaadan parivesh main vitarit kar diya jaataa he.
 software prashikshan or sahaayta mahatvapurn he, kyonki software pariyojnaon kaa ek bada pratishat is kaaran se asafal ho jaataa he ki developers yah nahin samaj pate hai ki yadi sanstha main koi bhi vyakti software kaa antim upyog nahin karata he to ise viksit karne vaale dal dwaara software ko taiyaar karne main lage samay or yojana kaa koi mahatva nahin rahata he. log aksar parivartan ke pratirodhi hote hai or ek aparichit kshetra main jokhim uthaane se bachate hai. isliye pariniojan charan ke hissaa ke rup main, aapke software ke nae graahakon ke liye prashikshan kakshaen yojit hona bahut aavashyak he.
 nae avishkrut samasyaon yaa nayi aavashyaktaaon kaa saamana karne ke liye software ke anurakshan or uske parivardhan main software ke aarambhik vikaas kii apeksha adhik samay lag saktaa he. apratyaashit samasya main sudhaar karne ke liye ek code ko jodna aavashyak he jo mul dizin main upyukt nahin bataata he yaa yah bhi ho saktaa he ki koi graahak adhik kaaryaatmakataa kaa anurodh kar rahaa ho or unke anurodh ko samyojit karne ke liye code ko joda ja saktaa he. isi charan main graahakon ke kol praapt hote hai or aap dekhte hai ki graahak dwaara prayog karne ke purv kya aapkaa pareekshan samasyaon ko dur karne ke liye vyapak rup se paryaapt tha. yadi anurakshan vaale charan kaa shram laagat purv charnon ke shram laagat kii apeksha 25% se adhik hota he, to yah sambhaavna hoti he ki kam se kam ek purv charan kii samgr gunvattta acchhee nahin he. us sthiti main, prabandhan ko anurakshan sambandhi laagat aniyantrit hone ke purv system ko punarnirmit karne ke vikalp par vichaar karnaa chahie.
 kisi vaastavik yaa mahasoos kiye gaye vishyon kii pahchaan karne vaale software kaa pareekshan karne vaale graahakon/kshetra dalon ke saath interphes dwaara jude rahane ke liye vikaas dalon ko anumati pradaan karne ke liye prakriya ke is charan main aksar bag trekking system upkaran pariniojit kiye jate hai. ye software upkaran, mukt strot or vanijyik license praapt donon, suchit vishyon ko apnaane, unki samiksha karne ke liye ek anukool karne yogya prakriya uplabdh karaate hai.
 samasyaon ke bahut pehle mahatvapurn vishyon par se pardaa hataane main shaamil logon kii madad karne ke liye punraritiya vikaas kisi software pariyojna ke liye prarambh main chote lekin hameshaa adhik bade hisson kii sanrachana nirdhaarit karti he. punraritiya prakriyaaon ko vanijyik developers ke dwaara adhik pasand kiya jaataa he, kyonki yah apni maang ko paribhaashit karne main asamarth graahak ke abhikalp lakshyon tak pahunchane kii sambhavyata kii anumati pradaan karata he.
 azeil software deveopment prakriyaaon kaa nirmaan punraritiya vikaas kii neenv par kiya jaataa he. ve us neenv main ek adhik halkaa, paaramparik drushtikonon kii apeksha adhik lok-kendrit drushtikon jodate hai. azeil prakriyaayein praathmik niyantran pranali ke rup main yojana kii apeksha feedback kaa upyog karti hai. feedback niyamit pareekshanon or viksit hone vaale software ke relies hone se prerit hota he.
 extream programing sarvshereshth gyaat punaravritiya prakriya he. pehle ke puraane "batch" prakriyaaon kii apeksha XP main, charnon ko atyadhik chote charnon main puraa kiya jaataa he. pratham charan un kadamon se hokar gujarata he, jisamen waterfol model ke prayek purna charan main lagane vaale maheenon or varshon kii tulana main ek din yaa ek saptaah lag saktaa he. sabase pehle, vyakti vikaas ke liye thos lakshya uplabdh karaane ke liye computarikrut pareekshanon ko ankit karata he. aglaa charan kuta lekhan he, jo sabhi pareekshanon ke biit jane or programmer dwaara or adhik pareekshan karne kii aavashyaktaa mahasoos nahin kiye jane ke baad purna ho jaataa he. dizin or sanrachana, rifracturing se utpann hoti he or kuta lekhan ke baad aati he. dizin unheen logon ke dwaara kiya jaataa he jo kuta lekhan karte hai. ke liye prabhavakaari bhang se prayukt yaa prardashit kii jaati he . is bindu par, visheshgy system ke agle sabase mahatvapurn bhaag ke liye pareekshan lekhan punah shuru karte hai.
 waterfol model ek prakriya ko darshata he, jahaan developers ko in kadamon kaa kram se paalan karnaa padataa he:
 jis prakaar dhaancha khadaa hone ke baad nirmaankarta ghar kii neenv nahin badalata he, prayek kadam ke puraa hone ke baad prakriya apane agle kadam main aage badhati jaati he.
 yah ek galat dharna he ki is prakriya main aarambhik kadamon main trutiyon ko sudhaarney kii koi vyavastha nahin he . vastav main, isi jagah aavashyaktaaon ke prabandhan kii baat saamne aati he, jisamen parivartan par niyantran shaamil he. prakriya ke aalochakon dwaara punrariti kii shuruyaat ke dwaara samasyaon ko dur karne main prativaadee tark mahatvapurn rup se ek badi hui laagat hoti he. yah bhi ek kaarak he jo supurdagee ke samay ko badha detaa he or ucch jokhim vaali pariyojnaon main bhi is prakriya ko teji se alokapriy bana detaa he.
 is drushtikon kaa prayog ucch jokhim vaali pariyojnaon, vishesh rup se raksha sambandhi bade anubandhon, main kiya jaataa he. waterfol main samasyaayein "vishesh rup se aavashyaktaaon ke vishleshan or prabandhan main aparipakv ejineering karypranaliyon" se utpann nahin hoti he. waterfol ko lagu karne vaali DOD-STD-2167 ke vinirdesh dar kii vifaltaa ke adhyayan ne yah darshaya he ki vishesh rup se agrim aavashyaktaaon ke sangrah main, koi pariyojna apni prakriya kaa jitna adhik dhyanpurvak paalan karata he, pariyojna dwaara varttman rup main aprayukt visheshtaon ko jari karne kii utani adhik sambhaavna hoti he.
 aksar kathith charan graahak or aapurtikarta ke bich samiksha kaa hissaa hote hai. vastav main, aapurtikarta jokhim main vikaas kar saktaa he or dizin kaa vikaas kar saktaa he, lekin use dizin ko mile ke pathar critical dizin rivyuu ko kam keemat main avashya bechnaa chahie. yah engring ke bhaar ko ejiniyaron se graahakon main visthapit kar detaa he, jinke anya kaushal hote hai.
 aupchaarik paddatiyaan aavashyaktaaon, vinirdesh or dizin ke str par software kii samasyaon ko hal karne ke ganitiya drushtikon hai. aupchaarik paddatiyon ke udhaaharan main bi-vidhi, petri nets, swachaalit prameya siddh karnaa, RAISE or VDM shaamil hai. vibhinn aupchaarik vinirdesh sanketan, jaise ki Z sanketan uplabdh hai. adhik saamaanya rup se, parimit avasthaa yantron ke system kii abhiklpana kar anuprayog kii pravriti kaa nirmaan evam abhipusht karne ke liye swachal praroop siddhant kaa prayog kiya ja saktaa he.
 parimit avasthaa yantra aadhaarit paddatiyaan nishpaadya software vinirdesh or paaramparik kutalekhan ke liye upmarg banane kii anumati pradaan karti he .
 aupchaarik paddatiyon kaa adhik sambhawatah vayaviki software main anuprayog kiya ja saktaa he, vishesh rup se jahaan software suraksha kii drushti se mahatvapurn he. software suraksha guarantee maanak, jaise ki DO178B vargeekaran ke ucchtam str par aupchaarik vidhiyon kii maang karti hai.
 anya sthanon main abhilakshya pratirodh bhashaa or vishesh rup se abhiklpanaaon ke nishpaadan, yadi vinirdesh nahin ho to bhi, kii anumati dene vaale model-chaalit sanrachana ke anuprayog ke saath-saath software deveopment kaa vargeekaran fail rahaa he.
 software ke vikaas main ek anya ubhartee hui pravriti tark ke kisi rup main ek vinirdesh lekhan karnaa or phir us tark kaa nishpaadan usee prakaar karnaa he jaise ki yah koi program ho. varnan tark par aadhaarit OWL bhashaa iskaa ek udahaaran he. taarkik rup main or taarkik rup se angreji ke kuch sanskarnon roopon kaa swatah pratichitran karne or tark ko sidhe nishpadit karne kaa kaam chal rahaa he. iske udhaaharan hai niyantrit prakritik angreji bhashaa or internet vyavsaay sambandhi tark, jinhen shabdkosh yaa vaakya vinyas ko niyantrit nahin karnaa padataa he. dwidishi angreji-taarkik pratichitran or tark kaa sidhe nishpaadan karne vaale tantra kii ek visheshta he ki vyavsaayik yaa vaigyanik str par unke dwaara angreji main unke parinaamon kii vyakhya karaayi ja sakati he.
