 ASP.NET ek web aplicoshan fremawork he jiskaa vikaas evam vipanan microsoft dwaara progaamar ko dynamik web site, web aplicoshan evam web sevaayen nirmit karne kii anumati hetu kiya gaya he. .NET fremawork ke sanskaran 1.0 ke saath ise pehali baar janvari 2002 main jari kiya gaya tha evam yah microsoft kii active sarwar pejiz praudyogiki kaa uttaradhikari he. ASP.NET kaa nirmaan kaaman laguage rantime par kiya gaya he jisase progamaron ko koi bhi sahaayak .NET bhashaa kaa prayog karte hue ASP.NET code likhane kii suvidha hoti he. ASP.NET SOAP ekstansion fremawork ASP.NET component ko SOAP sandeshon ko prakriyaarat karne kii suvidha pradaan karata he.
 internat soochna sevaayen 4.0 ke 1997 main jari hone ke uparaant microsoft ne ek nae web aplicoshan model hetu sambhaavnaaon ke anusandhan par kaary prarambh kiya jo ki ASP ke sambandh main saamaanya shikayaton, visheshkar prastut evam saamagree ke prathakkaran ke sambandh main, kaa niraakaran kar sake evam "clin" code likhane main sakasham ho. mark anders, prabandhak IIS team evam duk vishvavidhyalay se snaatak scot guthrie, jinhone 1997 main microsoft main kaarybhaar grahan kiya, ko is model ke nirdhaaran kii jimmedaari saunpi gai. anders evam guthrie dwaara do maah ke bheetar praarambhik dizin viksit kii gayi evam guthrie ne varsh 1997 main christmas ke avkaash ke dauraan praarambhik prototip ko kodebaddh kiya.
  praarambhik prototip ko "XSP" kaha gaya. varsh 2007 ke ek saakshaatkaar main guthrie ne bataaya ki "log hameshaa yah puuchhenge ki 'X' kaa kya matlab he. us samay iskaa vastav main koi matlab nahin tha. XML usase prarambh hua ; XSLT usase prarambh hua. prayek acchhee chiz X se prarambh hoti hui prathit hui isiliye hamane spashtatah iskaa yah naam rakhaa." XSP kaa praarambhik prototip java kaa prayog karte huye banaaya gaya tha, lekin jald hi kaaman laguage rantime ke sheersh par nae platform kaa nirmaan tay kiya gaya kyonki isane object oriented progaming vaataavaran, kachra ekathran evam anya visheshtaon kii suvidha pradaan kii, jo ki vaanchhneeya visheshtayen maani gayin jise microsoft kaa component object model plateform saport nahin karata. guthrie ne is nirnay ko ek "bada jokhim" qaraar diya kyonki unke naye web vikaas plateform kii safalataa CLR kii safalataa par nirbhar hogi jo ki XSP kii tarah vikaas kii aanrabhik avasthaa main he, yahan tak ki XSP team microsoft main pehali aisi team thi jisane CLR ko lakshya kiya tha.
 kaaman laguage rantime kii gati ke saath, XSP ko C# main punah karyanvit kiya gaya evam iskaa naam parivartit kar ASP+ rakhaa gaya kyonki is bindu tak active sarwar pejiz ke uttaradhikari ke rup main nae plateform ko dekha gaya evam ASP viksitakartaaon hetu aasaan migration maarg uplabdh karaane kaa iraadaa tha.
 phinics, arizona main 02 may 2000 ko yojit ASP conection sammelan main mark anders ne pehali baar ASP+ kaa pradarshan kiya. vistrut janata ke samaksh pradarshan evam ASP+ kaa aarambhik beta release 11 july 2000 ko orlando, plorida main yojit 2000 profeshnal dablopers sammelan main kiya gaya. bill gates kii mukhya prastuti ke dauraan, phujitsu ne ASP+ ko COBOL ke ek sanyugma ke rup main prayog kaa pradarshan kiya evam microsoft ke naye vijual basic .NET evam C# laguage kai anya laguage ke saath iske saport kii udghoshnaa kii gayi, iske alaava active state dwaara srujit interoparability tuls ke rup main paythan evam parl saport kii udghoshnaa kii gayi.
 varsh 2000 ke uttarardh main ".NET" brands kaa ek baar nishchay ho jane ke pashchaat ASP+ kaa naam badalakar ASP.NET rakhane kaa nirnay liya gaya. us varsh da MSDN sho main mark anders ne yah bataaya ki ".NET prayaas kai karkon ke bare main he, yah software ko ek seva ke rup main pradaan karne ke bare main he, yah XML evam web sevaaon ke bare main he evam yah internet main isaki kshamta anusaar vruddhi karne ke bare main he... ham iskaa naam .NET fremawork banane vaale anya plateform anshon ke saath laana chaahate hai."
 chaar varshon ke vikaas evam 2000 va 2001 main beta relies kii shrenee ke uparaant, ASP.NET 1.0 ko 05 janvari 2002 ko .NET fremawork ke sanskaran 1.0 ke ek bhaag ke rup main jari kiya gaya. release hone ke pehle hi, ASP.NET ke bare main darjanon kitaaben likhi gayin evam microsoft ne ise bade paimaane par web sevaaon hetu apane plateform ke rup main protsaahit kiya. guthrie ASP.NET hetu utpaada unit prabandhak bane evam vikaas jari rahaa tathaa 24 april 2003 ko windos sarwar 2003 ke ek bhaag ke rup main sanskaran 1.1 relies hua. is release main mobile upkarnon hetu ASP.NET ke saport main sudhaar par dhyaan kendrit kiya gaya.
 .NET pej, jinhen vidhivat rup se "web forms " ke rup main janaa jaataa he, anuprayog vikaas hetu mukhya nirmaan block hai. filon main web forms ".aspx " ekstansion ke saath upasthit rahete hai; ye filein vishesh rup se sthitij HTML markup ke saath sarwar said web control evam prayokta control ko paribhaashit karne vaale markup rakhti hai jisamen deblopers web pej hetu samast aavashyak sthitij evam gatij saamagree rakhte hai. iske atirikt dynamik code, jo ki sarwar par chalaata he, ko ek block ke bheetar ek pej main rakhaa ja saktaa he jo ki PHP, JSP tathaa ASP jaisi anya web vikaas praudyogikiyon ke samaan he, lekin data binding ke uddeshy ke alaava is tareeke ko saamanytah hatotsahit kiya jaataa he kyonki pej pradarshan ke samay isamen adhik colls kii aavashyaktaa hoti he.

 ASP.NET, vijited composit prastutikaran takanic kaa upyog karata he. sangrah ke dauraan, template file pravartan code main sangrahit ho jaati he joki mul template ko prastut karte hue ek control tree banaati he. paathya shabdic niyantran varg ke udaahranon main chalaa jaataa he, tathaa sarwar niyantran vishisht niyantran varg ke udaahranon dwaara prastut kiye jate hai. pravartan code prayokta dwaara likhit code ke saath mila hota he tathaa pej ke liye varg vishesh main parinaam detaa he. control tree ke mul ke anusaar pej doguna hota he.
 pej ke liye vaastavik anurodh ko kai charnon ke madhyam se sansaadhit kiya jaataa he. sabase pehle, pravartan charnon ke dauraan, pej ke varg kaa udhaaharan banaaya jaataa he tathaa pravartan code ko karyanvit kiya jaataa he. yah praarambhik control tree banaata he jiskaa ab baad ke charnon main pej kii vidhiyon dwaara visheshrup se parichaalan kiya jaataa he. chunki tree main prayek node, varg ke udhaaharan ke rup main prastut ek control hota he, jo code tree kii sanrachana ko badal sakane ke saath-saath ekal node ke gun-dharm/vidhiyon main bhi her-pher kar saktaa he. antat:, prastutikaran ke charan ke dauraan, visitor kii vidhiyon kaa prayog karke prayek node ko swayan prastutikaran karne kaa nirdesh dekar, tree main prayek node dekhne ke liye ek visitor kaa upyog kiya jaataa he. parinamswaroop HTML output graahak ko bhej diya jaataa he.
 anurodh ko process karne ke baad prishth shrenee kaa udhaaharan tathaa purna niyantran vriksh alag kar diya jaataa he. yah saamanytah nausikhia ASP.NET programmeron ke liye sanshay kaa strot hota he jo ki shrenee udhaaharan sadasyon par nirbhar hote hai jo prayek prishth anurodh/pratikriya chakra ke saath kho jate hai.
