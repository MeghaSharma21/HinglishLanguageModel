 si ek saamaanya upyog main aane vaali computer kii programmen bhashaa he. iskaa vikaas denis richi ne bell telephone prayogshaala main san 1972 main kiya tha jiskaa uddeshy unics sanchaalan tantra kaa nirmaan karnaa tha.
 is samay 'si' pehali yaa duusari sarvaadhik lokapriy programing bhashaa he. yah bhashaa vibhinn software flatformon par bahutayat main upyog kii jaati he. shaayad hi koi computer-platform ho jiske liye si kaa compiler uplabdh na ho. si++, java, si# aadi anek programmen bhashaaon par si bhashaa kaa gaharaa prabhaav dekha ja saktaa he.
 san 1960 main cambridge vishvavidhyalay ne ek computer programing bhashaa kaa vikaas kiya jise unhone BASIC COMBINED PROGRAMMING LANGUAGE naam diya. ise saamaanya bol-chaal kii bhashaa main bi kaha gaya. ’bi’ bhashaa ko san 1972 main bell prayogshaala main computer vaigyanik danish richi dwaara sanshodhit kiya gaya. ’si’ programing bhashaa ’bi’ programing bhashaa kaa hi sanshodhit rup he. ’si’ ko unics oparating system or dos oparating system dono main prayog kiya ja saktaa he, antar maatr compiler kaa hota he. unics oparating system ’si’ main likhaa gaya oparating sisstom he. yah visheshat: ’si’ ko prayog karne ke liye hi banaaya gaya he at: adhiktar ’si’ kaa prayog unics oparating system par hi kiya gaya he.
 si-bhashaa maamuli antar ke saath kai upbhaashaon ke rup main milti he. america rashtriya maanak santhaan  dwaara viksit ANSI C ko adhiktar maanak mana jaataa he.

 uparokt visheshtao ke kaaran hi ’si’ ek atyadhik lokapriy computer programing bhashaa he.
 isse pehle ham yah jaan chuke hai koi bhi char sanganak kii memory main kis tarah se sangrakshit hota he. jahaan yah char sangrakshit hota he uskaa ek nishchit pataa bhi hota he jo yah bataata he ki char kaa maan memory main kahan sangrakshit he. is pate ko hi pointer kehete hai. C/C++ programmen bhashaa main yah suvidha bhi hoti he ki kisi char kaa pataa gyaat kiya ja saken . C/C++ programmen bhashaa main bhi char kaa pataa jaanane ke liye & kaa upyog karte hai. jaise ki agar koi purnank char x he to x kaa pataa &x se mil jaayegaa. jis prakaar purnank, akshar, vaastavik sankhya ityadi ko char main sangrakshit kiya jaataa hai usee tarah kisi char ke pattey ko bhi. iske liye ek nayaa datatip hota he jo pataa sangrakshit karne ke kaam aata he jis tarah se purnank sangrakshit karne ke liye int datatype kaa upyog hota he. kisi purnank char kaa pataa sangrakshit karne ke liye int* datatype kaa upyog karte hai. isi tarah akshar char kaa pataa sangrakshit karne ke liye char* datatype kaa upyog karte hai. neeche ek chota sa udhaaharan yah diya gaya he jisamen ek char main duusare char kaa pataa sangrakshit karte hai.
 yahan pehle ek purnank char x paribhaashit kiya gaya he, phir p aisa char paribhaashit kiya he jo kisi purnank kaa pataa sangrakshit karata he. phir p char main x kaa pataa daal diya he. pataa →	0	1	2	3	4memory → 	10000111	11100101	00100110	0000101 	01100101 	. . .     ↑ p = &x = 3	int x	ab char ek purnank pointer p he jisamen char x kaa pataa sangrakshit he arthath p ko likhavane par x kaa pataa likhaa jaayegaa. yadi hamen yah jaananaa he ki p main jis memory kaa pataa likhaa hua us memory par kya sangrakshit he to *p kaa upyog karte hai karne ke liye hame ek main function avashya likhnaa hota he. kyonki ’si’ compiler kisi bhi program ko nishpadit karnaa main function se aarambh karata he. ek sanchika athwa ek program main ek se adhik main function nahin ho sakate.
 yah ek prayogkarta dwaara paribhaashit function he. main function ko ’{’ koshthak dwaara aarambh kiya jaataa he. program sanchika ke nishpaadan ke samay ’{’ yah bataata he ki nishpaadan yahan se aarambh karnaa he. isi prakaar ’}’ yah bataata he ki nishpaadan yahan samaapt hona he. ek program main main function to ek hi rahata he parantu anya function kaa prayog kiya ja saktaa he. prayek function ke liye { or }ke madhya upaprogram diya jaataa he. prayek nirdesh kaa ant semikalan ’;’ dwaara hona aavashyak he.
